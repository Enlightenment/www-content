Class Efl.Ui.Spin

A Spin. This is a widget which allows the user to increase or decrease a numeric value using arrow buttons. It's a basic type of widget for choosing and displaying values.

FormatString

A user-provided, string used to format the numerical value. For example, "%1.2f meters", "%.0%%" or "%d items".

This is the simplest formatting mechanism, working pretty much like printf.

Different format specifiers (the character after the %) are available, depending on the type used. Use Efl.Ui.FormatStringType.Simple for simple numerical values and Efl.Ui.FormatStringType.Time for time and date values. For instance, %d means "integer" when the first type is used, but it means "day of the month as a decimal number" in the second.

Pass NULL to disable this mechanism.

FormatValues

User-provided list of values which are to be rendered using specific text strings. This is more convenient to use than Efl.Ui.IFormat.FormatFunc and is perfectly suited for cases where the strings make more sense than the numerical values. For example, weekday names ("Monday", "Tuesday", ...) are friendlier than numbers 1 to 7. If a value is not found in the list, the other mechanisms will be tried according to their priorities. List members do not need to be in any particular order. They are sorted internally for performance reasons.

RangeLimits

Set the minimum and maximum values for given range widget. If the current value is less than min, it will be updated to min. If it is bigger then max, will be updated to max. The resulting value can be obtained with Efl.Ui.IRangeDisplay.GetRangeValue().

The default minimum and maximum values may be different for each class.

Note: maximum must be greater than minimum, otherwise behavior is undefined.

Methods

ApplyFormattedValue()

Internal method to be implemented by widgets including this mixin. The mixin will call this method to signal the widget that the formatting has changed and therefore the current value should be converted and rendered again. Widgets must typically call Efl.Ui.Format.formatted_value_get and display the returned string. This is something they are already doing (whenever the value changes, for example) so there should be no extra code written to implement this method.

Declaration
protected virtual void ApplyFormattedValue()

GetDecimalPlaces()

Internal method to be used by widgets including this mixin. It can only be used when a Efl.Ui.IFormat.GetFormatString(out System.String, out Efl.Ui.FormatStringType) has been supplied, and it returns the number of decimal places that the format string will produce for floating point values. For example, "%.2f" returns 2, and "%d" returns 0;
