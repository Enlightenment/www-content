PrecisePosition

Position where the event happened, relative to the window. This position is in floating point values, for more precise coordinates, in subpixels. Note that many input devices are unable to give better precision than a single pixel, so this may be equal to Efl.Input.Pointer.Position.

See also Efl.Input.Pointer.Position.

PreviousPosition

Position of the previous event, valid for move events. Relative to the window. May be equal to Efl.Input.Pointer.Position (by default).

This position, in integers, is an approximation of Efl.Input.Pointer.GetValue(Efl.Input.Value)(previous_x), Efl.Input.Pointer.GetValue(Efl.Input.Value)(previous_y). Use Efl.Input.Pointer.PreviousPosition if you need simple pixel positions, but prefer the generic interface if you need precise coordinates.

GetValue(Value)

Represents a generic value for this event. Refer to the documentation of Efl.Input.Value for each value's meaning, type and range. Call Efl.Input.Pointer.GetValueHas(Efl.Input.Value) to determine whether the returned value is valid or not for this event.

Most values are precise floating point values, usually in pixels, radians, or in a range of [-1, 1] or [0, 1]. Some values are discrete values (integers) and thus should preferably be queried with the other methods of this class.

This is a BETA method. It can be modified or removed in the future. Do not use it for product development.
