===== Description =====

%%Connects to a remote server offering an easy to use, buffered I/O.%%

%%The simple dialer is based on %%[[:develop:efl:auto:class:efl:net:socket:simple|Efl.Net.Socket.Simple]]%%, that encapsulates an actual %%[[:develop:efl:auto:interface:efl:net:socket|Efl.Net.Socket]]%%, and uses it with an %%[[:develop:efl:auto:class:efl:io:buffered_stream|Efl.Io.Buffered_Stream]]%%, which creates an input %%[[:develop:efl:auto:class:efl:io:queue|Efl.Io.Queue]]%%, an output %%[[:develop:efl:auto:class:efl:io:queue|Efl.Io.Queue]]%% and these are linked using a receiver and a sender %%[[:develop:efl:auto:class:efl:io:copier|Efl.Io.Copier]]%%.%%

%%The idea is that unlike traditional %%[[:develop:efl:auto:interface:efl:net:socket|Efl.Net.Socket]]%% that will attempt to write directly to socket and thus may take less data than requested, this one will keep the pending data in its own buffer, feeding to the actual socket when it %%[[:develop:efl:auto:interface:efl:io:writer:property:can_write|Efl.Io.Writer.can_write]]%%. That makes its operation much simpler as %%[[:develop:efl:auto:interface:efl:io:writer:method:write|Efl.Io.Writer.write]]%% will always take the full data -- allows "write and forget", if unlimited (see %%[[:develop:efl:auto:class:efl:io:buffered_stream:property:max_queue_size_output|Efl.Io.Buffered_Stream.max_queue_size_output]]%%).%%

%%Reading is also much simpler since received data is kept in an %%[[:develop:efl:auto:class:efl:io:queue|Efl.Io.Queue]]%%, thus its size can be queried with %%[[:develop:efl:auto:class:efl:io:buffered_stream:property:pending_read|Efl.Io.Buffered_Stream.pending_read]]%% and read with %%[[:develop:efl:auto:interface:efl:io:reader:method:read|Efl.Io.Reader.read]]%% or peeked with %%[[:develop:efl:auto:class:efl:io:buffered_stream:property:slice|Efl.Io.Buffered_Stream.slice]]%%, then discarded with %%[[:develop:efl:auto:class:efl:io:buffered_stream:method:discard|Efl.Io.Buffered_Stream.discard]]%% or %%[[:develop:efl:auto:class:efl:io:buffered_stream:method:clear|Efl.Io.Buffered_Stream.clear]]%%.%%

%%Then when waiting for a complete message, just peek at its contents, if not complete do nothing, if complete then either %%[[:develop:efl:auto:interface:efl:io:reader:method:read|Efl.Io.Reader.read]]%% to get a copy or manipulate a read-only reference from %%[[:develop:efl:auto:class:efl:io:buffered_stream:property:slice|Efl.Io.Buffered_Stream.slice]]%% and then %%[[:develop:efl:auto:class:efl:io:buffered_stream:method:discard|Efl.Io.Buffered_Stream.discard]]%%%%

%%The actual dialer is created using the class given as the constructor property %%[[:develop:efl:auto:class:efl:net:dialer:simple:property:inner_class|Efl.Net.Dialer.Simple.inner_class]]%% and can be retrieved with %%[[:develop:efl:auto:class:efl:io:buffered_stream:property:inner_io|Efl.Io.Buffered_Stream.inner_io]]%%, which should be used with care, like extra configuration before %%[[:develop:efl:auto:interface:efl:net:dialer:method:dial|Efl.Net.Dialer.dial]]%% is called.%%

%%If your object class requires some constructor-only properties to be set prior to %%[[:develop:efl:auto:class:efl:object:method:finalize|Efl.Object.finalize]]%%, then use %%[[:develop:efl:auto:class:efl:io:buffered_stream:property:inner_io|Efl.Io.Buffered_Stream.inner_io]]%% directly providing an already created dialer.%%
